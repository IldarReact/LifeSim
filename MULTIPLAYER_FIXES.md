# Исправления мультиплеера

## Проблемы, которые были исправлены:

### 1. ✅ Неправильный подсчёт игроков
**Было**: Awareness state не инициализировался, игроки не отображались
**Стало**: 
- При подключении автоматически устанавливается имя, цвет и статус
- Сохраняется ID локального клиента для правильной идентификации
- В списке показывается "(вы)" рядом с вашим ником

### 2. ✅ Ожидание обнуляется при перезагрузке
**Было**: Состояние `isWaiting` хранилось локально в React state
**Стало**:
- Статус готовности синхронизируется через Y.js Awareness
- При перезагрузке страницы статус восстанавливается
- Используется `setLocalState` вместо `setLocalStateField` для корректного обновления

### 3. ✅ Завершение хода не синхронизировано
**Было**: Каждый игрок вызывал `nextTurn()` независимо
**Стало**:
- Используется общая Y.Map "turnSync" для координации
- Когда все готовы, один игрок триггерит `advance: true`
- Все игроки подписаны на это событие и переходят ход одновременно
- После перехода флаг сбрасывается

## Ключевые функции:

### `initMultiplayer(roomId?)`
- Создаёт Y.Doc и WebRTC provider
- Инициализирует Awareness с дефолтными данными
- Сохраняет ID локального клиента

### `setPlayerReady(isReady: boolean)`
- Обновляет статус готовности в Awareness
- Синхронизируется со всеми игроками

### `subscribeToReadyStatus(callback)`
- Подписка на изменения статуса игроков
- Возвращает `readyCount`, `totalPlayers`, `allReady`

### `triggerTurnAdvance()`
- Устанавливает флаг `advance: true` в Y.Map
- Триггерит переход хода у всех игроков

### `syncTurnAdvance(callback)`
- Подписка на событие перехода хода
- Вызывает callback когда нужно перейти ход
