# Как тестировать мультиплеер локально

## Проблема
Мультиплеер работает через WebRTC и требует, чтобы игроки были в одной комнате. Но как это протестировать на одном компьютере без деплоя?

## Решение: Несколько окон браузера

### Способ 1: Обычные окна (Простой)
1. Запусти dev-сервер: `pnpm dev`
2. Открой игру в первом окне: `http://localhost:3000`
3. Создай персонажа и нажми кнопку **"Кооп"** (или "Создать лобби")
4. Скопируй URL из адресной строки (он будет содержать `?room=XXXXX`)
5. Открой **новое обычное окно** браузера (Ctrl+N)
6. Вставь скопированный URL в адресную строку
7. Создай второго персонажа

**Важно**: Оба окна будут использовать одно и то же хранилище (localStorage), поэтому состояние игры может конфликтовать!

### Способ 2: Режим инкогнито (Рекомендуется)
1. Запусти dev-сервер: `pnpm dev`
2. **Окно 1 (Хост)**: Открой `http://localhost:3000` в обычном окне
3. Создай персонажа, нажми "Кооп", скопируй URL
4. **Окно 2 (Гость)**: Открой **новое окно в режиме инкогнито** (Ctrl+Shift+N)
5. Вставь URL с room ID
6. Создай второго персонажа

**Преимущество**: Инкогнито использует отдельное хранилище, поэтому у каждого игрока будет свой независимый стейт.

### Способ 3: Разные браузеры (Самый надёжный)
1. Запусти dev-сервер: `pnpm dev`
2. **Chrome**: Открой игру, создай персонажа, нажми "Кооп", скопируй URL
3. **Firefox/Edge**: Открой скопированный URL, создай второго персонажа

**Преимущество**: Полностью изолированные среды, как у разных пользователей.

### Способ 4: Разные профили браузера (Продвинутый)
1. Создай новый профиль в Chrome (Settings → Manage Profiles → Add)
2. Запусти два окна Chrome с разными профилями
3. В каждом открой игру

**Преимущество**: Можно держать несколько профилей для постоянного тестирования.

## Что проверять

### Синхронизация хода
1. Оба игрока создают персонажей
2. Игрок 1 нажимает "Завершить ход"
   - ✅ Кнопка меняется на "Ожидание (1/2)"
   - ✅ В HUD у Игрока 1 появляется галочка
3. Игрок 2 видит:
   - ✅ В HUD у Игрока 1 появилась галочка
   - ✅ Счётчик показывает "1/2"
4. Игрок 2 нажимает "Завершить ход"
   - ✅ У обоих игроков автоматически переходит ход
   - ✅ Галочки сбрасываются
   - ✅ Квартал увеличивается

### Найм игроков в бизнес
1. Игрок 1 открывает бизнес
2. Игрок 1 идёт в "Управление персоналом"
3. Переключается на вкладку "Онлайн игроки"
   - ✅ Видит Игрока 2 в списке
4. Нанимает Игрока 2
   - ✅ Игрок 2 появляется в списке сотрудников

### Отключение игрока
1. Закрой окно Игрока 2
2. Проверь HUD у Игрока 1:
   - ✅ Счётчик меняется на "1 онлайн"
   - ✅ Игрок 2 исчезает из списка

## Частые проблемы

### "Бесконечная загрузка у гостя"
**Причина**: Гость не создал персонажа или застрял на экране setup.
**Решение**: Убедись, что оба игрока прошли setup и находятся в режиме "playing".

### "Хост сразу переходит ход"
**Причина**: Игра думает, что хост один (total = 1).
**Решение**: Проверь, что оба окна подключены к одной комнате (одинаковый room ID в URL).

### "Игроки не видят друг друга"
**Причина**: WebRTC не установил соединение.
**Решение**: 
- Проверь консоль браузера на ошибки
- Убедись, что signaling сервер доступен (`wss://y-webrtc-signal.fly.dev`)
- Попробуй перезагрузить оба окна

### "Состояние игры конфликтует"
**Причина**: Оба окна используют один localStorage.
**Решение**: Используй инкогнито или разные браузеры.

## Дебаг

Открой консоль браузера (F12) и проверь:
```javascript
// Проверить, подключен ли мультиплеер
console.log(window.location.search) // Должен содержать ?room=...

// Посмотреть список игроков (в консоли первого окна)
// Импортируй функцию в компоненте и вызови
getOnlinePlayers()
```

## Деплой для реального тестирования

Если нужно протестировать с реальным другом:
1. Задеплой на Vercel: `vercel --prod`
2. Отправь другу ссылку с room ID
3. Оба подключитесь к одной комнате

---

**Итог**: Для быстрого тестирования используй **инкогнито** или **разные браузеры**. Это самый простой и надёжный способ эмулировать двух игроков на одном компьютере.
