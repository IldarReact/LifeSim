# LifeSim - AI Project Memory

Это файл-память проекта LifeSim для AI-ассистента. Здесь собраны все успешные и неуспешные практики, архитектурные решения и рекомендации.

---

## 🟢 РАБОЧИЕ ПОДХОДЫ (высокий приоритет)

### Технологический стек
- **Next.js 16.0.0** + **React 19.2.0** + **TypeScript 5** — стабильная связка для SSR и клиентских компонентов
- **pnpm** — менеджер пакетов (используется вместо npm/yarn)
- **Tailwind CSS 4.1.9** — единственный источник стилей в проекте для быстрой разработки UI
- **Zustand 5.0.8** с persist middleware — глобальное состояние игры с автосохранением в localStorage
- **Recharts 2.15.4** — графики и визуализация данных (история игрока, статистика)
- **Radix UI** — headless компоненты для доступности и кастомизации
- **Framer Motion 12.23.24** — анимации и transitions
- **Zod 3.25.76** — валидация данных и type-safe схемы
- **Vitest 4.0.13** — юнит-тесты для бизнес-логики

### Архитектура проекта
**Feature-Sliced Design (FSD) подход:**
```
/core          — типы, модели, бизнес-логика (calculations)
/features      — фичи приложения (activities, events, gameplay, setup, reports, end)
/shared        — переиспользуемые UI компоненты и утилиты
/app           — Next.js App Router (page.tsx, layout.tsx)
```

**Строгие границы модулей (ESLint boundaries plugin):**
- `shared` НЕ может импортировать из `app`, `features`, `store`
- `features` НЕ может импортировать из `app`
- `lib` должен быть чистым и независимым

### Типизация и контракты данных
- **Strict TypeScript** (`strict: true` в tsconfig.json)
- **Запрет на `any`** через ESLint правило `@typescript-eslint/no-explicit-any: error`
- Все типы определены в `core/types.ts` (GameState, PlayerState, Asset, Debt, CountryEconomy и т.д.)
- Использование Type Aliases для обратной совместимости (например, `type Country = CountryEconomy`)

### Бизнес-логика
- Все расчёты изолированы в `core/lib/calculations/` (чистые функции)
- Каждый расчёт в отдельном файле с тестами (например, `calculateQuarterlyIncome.ts` + `calculateQuarterlyIncome.test.ts`)
- Zustand store (`core/model/game-store.ts`) оркеструет вызовы расчётов в методе `nextTurn()`

### UI/UX паттерны
- **"use client"** директива для всех интерактивных компонентов
- **Radix Dialog** для модальных окон (события, отчёты)
- **AnimatePresence** от Framer Motion для плавных переходов между экранами
- **Lucide React** для иконок
- **next-themes** для dark/light режима

### Git workflow
- **Husky** для pre-commit хуков
- **lint-staged** — автоматический ESLint + Prettier перед коммитом
- Prettier конфигурация: `semi: false`, `singleQuote: false`, `trailingComma: "es5"`

## 🟡 РАБОТАЕТ, НО С ОГОВОРКАМИ (средний приоритет)

### Recharts и темизация
- Recharts работает, но требует **ручной настройки цветов** для dark/light темы
- **Проблема (2025-11-22):** Цвета графиков не переключались автоматически при смене темы
- **Решение:** Использование `useTheme()` хука и явная передача цветов в компоненты Recharts
- **Оговорка:** Код становится более verbose, требуется дублирование логики цветов

### Комментарии в коде
- **Практика (2025-11-22):** Минимизация комментариев, оставляем только:
  - Навигационные блоки (например, `// 1. Global Events`, `// 2. Economy`)
  - Объяснения неочевидной логики (например, обработка событий Recharts)
- **Оговорка:** Баланс между читаемостью и избыточностью — субъективен

## 🧠 КЛЮЧЕВЫЕ ОСОБЕННОСТИ ПРОЕКТА

### Игровая механика
- **Квартальная система:** 1 ход = 1 квартал, 4 хода = 1 год
- **Энергия игрока:** Ограниченный ресурс (0-100), тратится на активности
- **Мультистрановая экономика:** Разные страны с разными архетипами (rich_resource, growing_stable и т.д.)
- **Архетипы персонажей:** investor, specialist, entrepreneur, worker, indebted
- **Случайные события:** Глобальные (влияют на экономику) и персональные (влияют на игрока)

### Расчётная система
Все расчёты происходят в методе `nextTurn()` в строгом порядке:
1. Глобальные события
2. Экономика стран
3. Доход игрока
4. Активы и долги
5. Налоги
6. Расходы
7. Баланс наличности
8. Личная жизнь (счастье, здоровье, отношения)
9. Случайные персональные события
10. Обновление истории (каждый год)

### Персистентность
- **Zustand persist middleware** с ключом `life-sim-save-v2`
- Автосохранение в localStorage после каждого действия
- Возможность продолжить игру после перезагрузки страницы

### Тестирование
- **Vitest** для юнит-тестов бизнес-логики
- Тесты находятся рядом с тестируемым кодом (например, `calculations.test.ts`)
- Используется `seedrandom` для детерминированных случайных чисел в тестах

### Линтинг и форматирование
- **ESLint** с `next/core-web-vitals` и `prettier` конфигами
- **eslint-plugin-boundaries** для контроля архитектурных границ
- **Prettier** для автоформатирования
- **Husky + lint-staged** для автоматической проверки перед коммитом

### Специфичные решения
- **Отсутствие отрицательного баланса:** При `cash < 0` баланс обнуляется, но выводится предупреждение
- **Модальные окна для событий:** `pendingEventNotification` в store + `EventModal` компонент
- **Годовые отчёты:** Генерируются каждые 4 хода и сохраняются в `history[]`
- **Активности с энергией:** Каждая активность требует энергии, которая восстанавливается в конце квартала
